# Inbox

Use this folder as a low-friction entry point. You can place a short Markdown
note here or ask Codex to capture a thought for you. Rough wording is fine.

Good inbox items include:

- a mathematical question you do not yet know how to formulate;
- a URL or citation to inspect;
- a reference to a handwritten note or scan;
- a fragment from a conversation;
- a request such as “turn this into a precise conjecture.”

Codex should preserve the original, propose a destination, and create or update
the appropriate durable note. Once the result is linked, move the original to
`inbox/archive/YYYY-MM/`. Never permanently delete it.

Start from `meta/templates/inbox.md` when a structured capture is helpful. You
do not need to understand YAML; Codex can fill the fields after confirming the
title, date, and selected language.
