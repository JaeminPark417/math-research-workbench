# 처음 시작하기: 한 단계씩 따라 하기

이 안내서는 브라우저에서 ChatGPT를 사용해 본 적은 있지만 Git, 터미널,
Markdown, Codex는 처음인 분을 기준으로 작성했습니다. 이 도구들을 미리
공부하지 않아도 시작할 수 있습니다.

## 이 안내서에서 자주 쓰는 일곱 용어

- **폴더(folder)**: 컴퓨터 안에서 파일을 모아 두는 상자입니다. 종이 문서를
  넣는 서류함과 비슷합니다.
- **Markdown**: 제목, 링크, 수식을 나타내는 간단한 기호가 조금 추가된 일반
  텍스트입니다. 파일 이름은 `.md`로 끝납니다.
- **vault(볼트)**: Obsidian에서 노트 폴더를 부르는 이름입니다. 이 프로젝트에서는
  workbench 폴더와 Obsidian vault가 같은 폴더입니다.
- **Git/GitHub**: 텍스트 파일의 변경 이력과 온라인 사본을 보관하기 위한 선택
  기능입니다. Google Drive나 Dropbox와는 다른 서비스입니다.
- **TeX/LaTeX**: `.tex` 원고를 조판하여 PDF로 만드는 시스템입니다. 일반 노트를
  쓰거나 Obsidian에서 수식을 보는 데에는 필요하지 않습니다.
- **Claude Code/Claude**: 두 번째 AI 검토에 선택적으로 사용할 수 있는 Anthropic의
  터미널 도구와 AI 서비스입니다. OpenAI, ChatGPT, Codex와는 별개입니다.
- **앱 내 Browser**: 호환되는 스킬이 사용할 수 있는 데스크톱 앱 안의 선택형
  브라우저 로그인 공간(profile)입니다. 평소 사용하는 웹 브라우저와 분리되어
  있습니다.

## 1. 안전한 위치 고르기

workbench의 텍스트 파일은 Git으로 관리할 수 있습니다. 같은 폴더를 클라우드
동기화 프로그램이 동시에 관리하게 하지 마세요. 두 시스템이 같은 파일의 이름을
바꾸거나 합치려고 하면 충돌이 생길 수 있습니다.

사용자 홈 폴더 바로 아래에 `MathResearch`라는 폴더를 만들고, 그 안에
workbench를 두는 것을 권합니다. 컴퓨터가 바탕 화면이나 문서 폴더를 자동으로
동기화한다면 그 위치는 피하세요. 위치를 잘 모르겠다면 초기 설정을 계속하기 전에
Codex가 확인해 줍니다.

Google Drive, Dropbox, OneDrive 같은 서비스는 PDF, 스캔본, 발표 자료 등 큰
파일을 보관하는 데 여전히 유용합니다. 초기 설정에서 이런 파일만 넣을 별도 폴더를
나중에 연결할 수 있습니다.

## 2. workbench 받기

### 권장 방법: 릴리스 ZIP 파일

1. [최신 릴리스 페이지](https://github.com/JaeminPark417/math-research-workbench/releases/latest)를
   엽니다.
2. **Assets**에서 Math Research Workbench 이름이 붙은 ZIP 파일을 내려받습니다.
3. ZIP 파일을 위에서 고른 안전한 위치로 옮깁니다.
4. 압축을 풉니다.
5. 압축을 푼 폴더를 열어 `README.md`와 `AGENTS.md`가 보이는지 확인합니다.
   바로 그 폴더를 Codex에서 열면 됩니다.

ZIP 파일은 특정 시점의 복사본일 뿐입니다. ZIP을 받았다고 해서 내 연구 폴더가
GitHub와 연결되거나 자동으로 백업되는 것은 아닙니다. 초기 설정 중에 비공개
GitHub 백업을 사용할지 물어보고, 원할 때 연결 과정을 안내합니다.

### 다른 방법: GitHub 템플릿으로 만들기

이미 GitHub 계정을 사용하고 있고, 개인 연구용 비공개 저장소를 만들고 싶을 때만
이 방법을 선택하세요.

1. 저장소 페이지에서 **Use this template** → **Create a new repository**를
   선택합니다.
2. 기밀 연구 내용을 드러내지 않는 저장소 이름을 정합니다.
3. **Private**를 선택합니다. 출판 전 연구를 공개 기본값인 채로 만들지 마세요.
4. 저장소를 만듭니다.
5. **Code** → **Open with GitHub Desktop**을 선택하고, 앞서 설명한 안전한
   비동기화 위치에 복제(clone)합니다. 이렇게 해야 컴퓨터의 폴더가 비공개
   저장소와 연결됩니다.
6. GitHub Desktop을 사용하고 싶지 않다면 권장 방법인 릴리스 ZIP을 사용하세요.
   새 비공개 저장소에서 ZIP을 내려받는 것만으로는 압축을 푼 폴더가 그 저장소와
   연결되지 않습니다.

템플릿으로 만든 저장소에는 원본 템플릿의 업데이트가 자동으로 들어오지 않습니다.
현재 절차는 GitHub의
[템플릿에서 저장소 만들기 안내](https://docs.github.com/en/repositories/creating-and-managing-repositories/creating-a-repository-from-a-template)에서
확인할 수 있습니다.

## 3. Codex 설치하고 폴더 열기

초보자에게 권하는 데스크톱 방식은 macOS와 Windows에서 사용할 수 있습니다.
Linux에는 현재 ChatGPT 데스크톱 앱이 없으므로, 이 workbench를 사용하려면
[Codex CLI](https://developers.openai.com/codex/cli/)와 터미널의 기본 사용법이
필요합니다. 이 절의 나머지는 쉬운 데스크톱 방식을 설명합니다. Linux에서는 공식
CLI 안내에 따라 설치한 뒤, 터미널에서 압축을 푼 workbench 폴더로 이동해 Codex를
시작하고 똑같이 `초기 설정을 시작해줘`라고 보내세요.

1. OpenAI 공식 사이트에서
   [ChatGPT 데스크톱 앱](https://chatgpt.com/download/)을 내려받습니다.
2. 일반 프로그램처럼 설치하고 로그인합니다.
3. 데스크톱 앱에서 **Codex**를 선택합니다.
4. 로컬 폴더 또는 프로젝트를 여는 메뉴를 선택합니다.
5. `AGENTS.md`가 들어 있는 workbench 폴더를 선택합니다.
6. 앱이 이 폴더를 신뢰할지 또는 로컬 접근을 허용할지 묻는다면, 화면에 표시된
   경로가 내가 고른 폴더가 맞는지 확인한 뒤에만 허용합니다.

로컬 폴더 사용법과 데스크톱 지원 현황에 대한 OpenAI의 현재 설명은
[ChatGPT Work and Codex](https://help.openai.com/en/articles/20001275)에서 볼 수
있습니다. macOS와 Windows에서는 Codex CLI를 따로 설치할 필요가 없습니다.

## 4. 첫 메시지 보내기

폴더를 열기만 해서는 대화가 시작되지 않습니다. Codex는 사용자가 메시지를
보낼 때까지 기다립니다. 다음 문장을 입력하고 전송하세요.

```text
초기 설정을 시작해줘
```

영어로 `Start setup`이라고 쓰거나 `$first-run`을 실행해도 됩니다.

답변을 저장하기 전에 Codex가 Python 3.9 이상 실행 환경을 확인합니다. 이미 있거나 Codex에
번들된 환경을 먼저 사용하며, 사용자가 Python 프로그래밍을 할 필요는 없습니다.
없다면 공식 사용자용 설치 프로그램의 의미를 설명하고, 열기·다운로드·실행 전에
각각 승인을 받습니다. 거절해도 일반 Markdown 작업은 가능하지만, 재개할 수 없는
부분 상태를 만들거나 재개된다고 약속하지 않습니다.

도우미 실행 환경이 준비된 뒤에는 도중에 앱을 닫아도 됩니다. 같은 폴더를 다시
열고 같은 메시지를 보내면 완료된 답변은 이 컴퓨터에만 기록되고, 초기 설정은
끝나지 않은 첫 질문부터 이어집니다.

## 5. 초기 설정 질문에 답하기

Codex는 한 번에 하나씩 질문합니다. 모든 질문에서 `아니요`나 `나중에`를
선택해도 괜찮습니다.

### 안내 언어

낯선 용어와 변경 내용을 Codex가 어떤 언어로 설명할지 고릅니다.

### GitHub 백업

GitHub는 텍스트 파일의 여러 버전을 보관할 수 있지만 필수는 아닙니다. 사용하기로
선택하면 다음 원칙을 따릅니다.

- 출판 전 연구 저장소는 기본적으로 비공개로 만듭니다.
- Codex는 브라우저 로그인이나 기기 인증을 사용하며, 비밀번호나 토큰을 채팅에
  붙여 넣으라고 요구하지 않습니다.
- 현재 연결은 민감한 주소 대신 가려진 안전 분류로만 설명합니다. 사용자가 대상
  소유자와 저장소 이름을 정하거나 승인한 뒤에는, 인증 정보가 없는 표준 주소와
  공개 범위를 무엇을 바꾸기 전에 보여 줍니다.
- 큰 PDF 파일은 GitHub 밖에 둡니다.

**GitHub는 자동 백업이 아닙니다.** 변경 사항을 커밋(commit)하고 푸시(push)한
뒤에야 GitHub에 나타납니다. Codex는 이 용어의 뜻을 설명하고, 포함될 파일을
보여 주고, 목적지가 비공개인지 확인한 뒤, 동기화할 때마다 먼저 허락을 받아야
합니다.

공개된 Math Research Workbench 저장소는 배포용 템플릿입니다. 내 비공개
연구를 그 저장소에 올리는 곳이 아닙니다.

### 외부 파일 저장소

Google Drive, Dropbox, OneDrive, 이미 사용 중인 다른 폴더, 컴퓨터에만 보관,
또는 나중에 설정 중 하나를 선택합니다. Codex는 선택한 폴더를 확인하고, 그 안에
하위 폴더를 만들기 전에 허락을 구합니다. PDF 같은 큰 파일만 그곳에 두고,
workbench 자체는 안전한 로컬 위치에 그대로 둡니다.

컴퓨터에만 보관하기를 선택하면 `files/` 폴더를 쓸 수 있지만, 이 폴더의 내용은
GitHub에 백업되지 않습니다.

### Obsidian

Obsidian은 선택 사항입니다. 사용하기로 하면 Codex가 설치 여부를 확인하고,
공식 설치 방법을 제시하며, 어떤 프로그램이든 설치하기 전에 허락을 받습니다.
그다음 **현재 workbench 폴더를 그대로** vault로 엽니다. 두 번째 복사본을 만들지
마세요.

처음에는 기본(core) 플러그인만 사용하세요. Codex는 실행되지 않는 상태로 함께 배포된 LaTeX 구분자 호환 플러그인을 별도 항목으로 제시하고, 접근 권한·공식 목록 미등록·업데이트 방식·테스트 절차를 설명한 뒤 설치 여부를 묻습니다. 제3자 커뮤니티 플러그인은 계속 선택 사항이며 Obsidian 공식 목록에서 한 번에 하나씩만 설치합니다. 자세한 내용은 [Obsidian 안내](docs/obsidian.ko.md)와 [Obsidian 플러그인 안내](docs/obsidian-plugins.ko.md)를 읽어 보세요.

### TeX

`나중에`, Overleaf, 컴퓨터에 직접 설치, 또는 사용하지 않음 중 하나를
고릅니다. 처음에는 브라우저에서 조판할 수 있는 Overleaf가 가장 간단합니다.
로컬 TeX를 선택하면 Codex가 먼저 이미 설치된 도구가 있는지 확인하고, 예상
다운로드 크기와 디스크 사용량을 확인할 수 있으면 보여 주며, 알 수 없으면
추측하지 않고 확인할 수 없다고 말합니다. 실행할 설치 명령을 정확히 보여 준 다음
허락을 받습니다. 설치에 실패하거나 나중으로 미뤄도 나머지 초기 설정은 계속할
수 있습니다.

### Claude 검토

Claude 검토는 선택 사항입니다. Anthropic의 터미널 도구인 Claude Code를 통해
별도의 AI 서비스에 두 번째 의견을 요청합니다. ChatGPT나 OpenAI 계정에 포함된
기능이 아니며, Claude Pro 또는 Max의 개인 직접 구독 로그인이 필요합니다. 이
초보자용 흐름은 Team, Enterprise, Console/API key, 클라우드 제공자, proxy, 별도
gateway를 사용하지 않습니다. Claude의 safe mode에서도 관리형 설정은 남을 수
있으므로, 관리형 정책이 발견되거나 안전하게 확인할 수 없으면 이 단계는 멈춥니다.
Anthropic의 최신 공식 [설치 안내](https://code.claude.com/docs/en/installation)와
[인증 안내](https://code.claude.com/docs/en/authentication)를 확인하세요.

`예`를 선택하면 Codex가 먼저 Claude Code가 이미 있는지 확인합니다. 설치가
필요하면 공식 출처와 실행할 정확한 작업을 설명하고 설치 승인을 받습니다.
Anthropic 로그인을 시작하는 일은 설치와 별개이므로 다시 승인받습니다. Claude
Code는 터미널에서 실행되지만, 대화형 터미널이나 열린 브라우저의 비밀번호,
passkey, MFA, OAuth 단계는 사용자가 직접 완료합니다. Codex는 그동안 멈추고 인증
화면을 검사하거나 캡처하지 않습니다. 인증 코드를 채팅에 붙여 넣지 말고, 이
초보자용 설정에서는 `claude setup-token`을 사용하지 않습니다.

터미널 명령을 이해할 필요는 없습니다. macOS에서는 Spotlight에서 **Terminal**을,
Windows에서는 시작 메뉴에서 **Terminal** 또는 **PowerShell**을 여세요. Codex가
입력할 정확한 `claude auth login` 명령을 보여 주고, 사용자가 비공개 로그인 화면을
직접 마칠 때까지 기다립니다.

로그인 뒤에는 사용자가 Claude를 safe mode로 열고 `/status`를 직접 입력합니다.
`Setting sources`만 확인한 다음 Claude를 닫고, `Enterprise managed settings`
출처가 있었는지만 답합니다. 나머지 화면을 채팅에 복사하지 마세요. 그런 출처가
없었다고 확인할 수 없으면 초기 설정과 이후의 각 검토를 진행하지 않습니다.

로그인 성공은 Claude를 사용할 준비가 되었다는 뜻일 뿐, 검토 자료 전송에 동의한
것은 아닙니다. `claude-review` 스킬은 검토할 때마다 Anthropic이 제공자임을 밝히고,
목적과 외부로 나갈 정확한 파일·diff·본문을 나열한 뒤 그 한 번의 검토에 대해
승인을 받아야 합니다. 거절해도 평소 Codex 작업은 계속할 수 있습니다. Claude의
검토 역시 AI 검토이며, 검증된 증명이 아닙니다.

### 앱 내 Browser의 ChatGPT

이 로그인은 선택 사항입니다. 번들된 `$pro-context-bundle`은 사용자가 명시적으로
요청한 Pro 상담을 위해 앱 내 Browser의 ChatGPT를 사용합니다. 해당 스킬을 쓸
계획이 없다면 `나중에`를 선택하세요.

앱 내 Browser는 지원되는 macOS 또는 Windows 데스크톱 환경에서만 사용할 수
있습니다. 제품 기능 제공 여부, 요금제, workspace 정책에 따라 보이지 않을 수
있으며, Linux에서는 사용할 수 없습니다. 일반 브라우저나 다른 앱의 로그인 상태와
분리된 로그인 공간(profile)이므로 기존 ChatGPT 로그인이 이어지지 않을 수
있습니다. OpenAI의 공식
[Browser 안내](https://help.openai.com/en/articles/20001277-using-the-built-in-browser-in-the-chatgpt-desktop-app)를 확인하세요.

ChatGPT가 로그인을 요구하면 Codex가 멈춥니다. 비밀번호, passkey, MFA 응답,
OAuth 코드는 사용자가 직접 입력하며, Codex는 인증 화면을 검사하거나 캡처하지
않습니다. 로그인은 Browser의 로그인 공간을 준비할 뿐 파일 업로드나 메시지 전송을
허용하지 않습니다. 호환 스킬이 무엇인가 보내기 전마다 Codex는 제공자, 목적,
외부로 나갈 정확한 파일이나 본문을 보여주고 그 한 번의 작업에 대해 승인을 받아야
합니다. 전송 뒤에는 전체 시간 제한 없이 답변을 기다리며, 상담 도중 사용자가
명시적으로 요청할 때만 `Answer now`를 누릅니다.

## 6. 초기 설정이 끝났는지 확인하기

Codex는 파일을 바꾸지 않는 안전한 점검을 실행한 뒤 다음 내용을 설명합니다.

- 준비가 끝난 항목
- 일부러 건너뛴 항목
- 나중으로 미룬 선택
- 선택을 나중에 바꾸는 방법

이어서 폴더 구성을 설명하고, 바로 써 볼 수 있는 몇 가지 첫 질문을 제안합니다.
나중에 설정 하나만 바꾸고 싶다면 다음처럼 말하면 됩니다.

```text
초기 설정을 다시 실행하되, Obsidian 설정만 도와줘.
```

## 호스팅 또는 클라우드 환경에서 열었다면

클라우드 checkout은 다른 컴퓨터에서 실행되는 복사본입니다. 내 PC에 Obsidian,
TeX, Claude Code를 설치하거나 데스크톱 앱의 Browser 로그인 공간을 준비할 수
없고, 내 컴퓨터의 클라우드 드라이브 폴더를 확인할 수도 없으며, 이 컴퓨터에서만
쓰는 설정을 저장할 수도 없습니다. 클라우드에 있는 Markdown 파일을 읽고 수정하는
일은 가능할 수 있지만, 초기 설정 도우미는 이 한계를 설명하고 로컬 데스크톱
설정으로 안내해야 합니다. 클라우드 작업만으로 내 PC의 초기 설정을 완료했다고
표시해서는 안 됩니다.

## 연구 내용을 넣기 전에 개인정보 확인하기

로컬 폴더를 열었다고 해서 Codex가 오프라인 프로그램이 되는 것은 아닙니다.
요청에 답하는 데 필요한 내용이 처리를 위해 OpenAI로 전송될 수 있습니다. 출판
전 심사 보고서, 학생이나 환자의 정보, 비공개 공동 연구, 수출 통제 자료 또는
그 밖의 제한된 내용을 사용하기 전에 소속 기관의 정책과 계정 또는 workspace
설정을 확인하세요. OpenAI의
[데이터 제어 안내](https://help.openai.com/en/articles/7730893-data-controls-faq)에는
ChatGPT 계정에서 사용할 수 있는 제어 기능이 설명되어 있습니다. 기관용 요금제는
기본값과 관리자 규칙이 다를 수 있습니다.

## 초기 설정이 몰래 하지 않는 일

초기 설정은 사용자에게 알리지 않고 다음 작업을 하지 않습니다.

- 프로그램 설치
- 연구 내용 공개
- 온라인 저장소 생성 또는 삭제
- Git 연결 주소(remote) 변경
- 현재 열려 있는 workspace 이동
- 외부 드라이브에 파일 쓰기
- 커뮤니티 플러그인 일괄 설치
- Claude Code 설치 또는 Anthropic 로그인 시작
- 외부로 나갈 내용을 새로 정확히 보여주고 그 한 번의 승인을 받지 않은 채 Claude나
  ChatGPT로 연구자료 전송
- 연구 파일 영구 삭제

다음 두 인증 경계는 승인받으면 할 수 있는 작업이 아니라 절대 지켜야 하는
원칙입니다. 초기 설정은 승인 여부와 무관하게 `claude setup-token`을 실행하지
않으며, Codex는 인증 화면을 검사하거나 캡처하지 않습니다. 인증은 항상 사용자가
직접 완료합니다.

평소 작업 흐름은 [일상 사용 가이드](docs/daily-workflow.ko.md)에서 이어서
확인할 수 있습니다. 새 배포판을 적용할 때는
[안전한 업데이트 안내](docs/updating.ko.md)를 먼저 읽으세요.
