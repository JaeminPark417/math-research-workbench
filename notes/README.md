# Notes

This folder holds reusable mathematical knowledge: definitions, standard
examples, proof techniques, notation guides, and explanations that are useful
across more than one project.

Organize notes in subject subfolders when a topic grows, for example
`notes/algebraic-geometry/` or `notes/probability/`. Use lowercase
`kebab-case` filenames.

Reusable notes may be plain Markdown without YAML frontmatter. Do not add an
extra YAML type for them; the supported workflow types are listed in
`meta/schemas.md`.

Each note should link to the verified sources and project research logs from
which it was synthesized. Clearly label conjectural material and open gaps.
An assistant-generated explanation remains a draft until a researcher checks
it.
